# walk3d 버그 수정 내역

## 1회차 수정 (4개 이슈)

### 1. 도로에 건물이 가로막는 문제
- **원인**: `ChunkGenerator.ts`의 도로 셀 스킵 조건이 `&&`(교차점만 스킵)로 되어 있어 도로 행·열 전체가 스킵되지 않았음
- **수정**: `&&` → `||` 로 변경 (`src/world/ChunkGenerator.ts`)
- **결과**: 청크당 4개 코너 셀에만 건물 배치 → 도로 행/열 완전 비워짐

### 2. 벤치 오브젝트가 너무 작은 문제
- **원인**: Kenney GLB 에셋 스케일이 `0.9`로 설정되어 실제로 매우 작게 보였음
- **수정**: 벤치 GLTF 스케일 `0.9` → `2.2` (`src/world/Props.ts`)

### 3. 고양이가 건물을 통과하는 문제
- **원인**: 건물 충돌 판정이 없었음
- **수정**:
  - `src/world/BuildingColliders.ts` 신규 생성 — 원통형 충돌 레지스트리
  - `src/world/ChunkMeshFactory.ts` — 청크 로드 시 건물 위치 등록, 청크 해제 시 삭제
  - `src/main.ts` — 매 프레임 `checkBuildingCollision()` 호출, X/Z 축 분리 슬라이딩 충돌 처리
- **충돌 반경**: 건물 중심으로부터 3.8 유닛

### 4. 경계 근처 아이템에 접근이 안 되는 문제 (1차 수정)
- **원인**: 아이템 스폰 범위가 청크 크기의 ±0.8배여서 잠긴 지역 경계 가까이 아이템이 생성됨
- **수정**: 스폰 범위 `0.8` → `0.55` (`src/world/ChunkGenerator.ts`)

## 2회차 수정 (5개 이슈)

### 5. 가로등이 너무 작고 방향이 부자연스러운 문제
- **원인**: GLTF 스케일 `1.8`이 너무 작았고, 팔(arm)이 한쪽으로 튀어나온 비대칭 디자인이라 무작위 회전 시 어색하게 보였음
- **수정**:
  - 가로등 스케일 `1.8` → `5.0` (`src/world/Props.ts`)
  - 폴백 `makeLamp()` 디자인을 팔 없는 대칭형(기둥 + 구형 전구)으로 변경

### 6. 코인 아이템이 눕혀서 회전하는 문제
- **원인**: `CylinderGeometry`가 기본적으로 Y축 방향(수평 원반)으로 생성됨, 토러스 링이 토성 형태로 보임
- **수정**: `disc.rotation.x = Math.PI / 2` 추가 + 토러스 제거 → 코인이 세워져 좌우로 회전 (`src/game/ItemSystem.ts`)

### 7. 나무 크기가 너무 작은 문제
- **원인**: GLTF 스케일 `2.5`가 건물 대비 작게 보였음
- **수정**: 나무 스케일 `2.5` → `4.0` (`src/world/Props.ts`)

### 8. 고양이가 건물 뒤에 있을 때 시야 방해 문제
- **원인**: 건물이 카메라와 고양이 사이에 있어도 그대로 렌더링됨
- **수정**:
  - `src/world/ChunkMeshFactory.ts` — 건물 메시에 `userData.occludable = true` 태그
  - `src/main.ts` — 매 프레임 카메라→고양이 방향으로 레이캐스트, occludable 태그 메시가 중간에 걸리면 투명도 `0.18`로 낮추고, 가리지 않게 되면 원래 투명도 복원

### 9. 가로등/소품이 도로 위에 스폰되는 문제
- **원인**: 소품 스폰 위치에 도로 밴드 체크가 없었음
- **수정**: 소품 생성 시 도로 중심 기준 ±7유닛 이내 위치 스킵 (`src/world/ChunkGenerator.ts`)

## 3회차 수정 (3개 이슈)

### 10. 잠긴 지역에 아이템이 여전히 스폰되는 문제 (근본 원인 수정)
- **원인**: 아이템/소품/건물이 `cx * CHUNK_SIZE`를 중심으로 ±range 생성됨. 게임의 청크 논리 경계는 `Math.floor(x / CHUNK_SIZE) = cx`로 `cx*32 ~ (cx+1)*32` 범위인데, 중심이 경계 시작점이라 음수 방향에서 아이템이 인접한 잠긴 청크 영역에 떨어짐 (예: cx=-2 → 아이템이 cx=-3 영역에 스폰)
- **수정**: 모든 생성 좌표 중심을 `cx * CHUNK_SIZE` → `(cx + 0.5) * CHUNK_SIZE`로 변경 (`src/world/ChunkGenerator.ts`)
- **세이브 처리**: `CURRENT_ITEM_SCHEMA_VERSION` 1 → 2 (기존 아이템 위치 무효화)

### 11. 도로/지형 비주얼이 너무 단순한 문제
- **원인**: 도로가 폭 3유닛의 단색 평면 2개로 이루어져 시각적으로 너무 좁고 단조로움. 지형도 단색 평면
- **수정**:
  - `src/world/RoadGrid.ts` 전면 재작성 — 총 폭 14유닛(도로 10 + 인도 2×2), 중앙선 점선, 가장자리 선 추가
  - `src/world/ChunkMeshFactory.ts` — 지형 ground를 ShaderMaterial 격자 패턴으로 교체 (지역 색상 유지)
  - 도로/건물/소품 중심 좌표도 `(cx + 0.5) * CHUNK_SIZE`로 통일하여 청크 내부 정렬

### 12. Z-fighting 현상
- **원인**: 지면(y=0.005)과 도로(y=0.008) 사이 Y 간격이 0.003유닛으로 너무 좁아 부동소수점 정밀도 한계로 깜빡임 발생
- **수정**:
  - 지면 y: `0.005` → `0.0`
  - 도로/인도 y: `0.008` → `0.02`
  - 가장자리 선: `y+0.002` → `y+0.008`
  - 중앙선 점선 y: `0.016` → `0.032`
  - 도로/인도/선 재질에 `polygonOffset: true` + `polygonOffsetFactor/Units` 적용 (`src/world/RoadGrid.ts`)
