## Phase 0 — 기획 · 아키텍처 수립 (2026-04-17)

> OMC의 `deep-interview` + `plan --consensus` 파이프라인을 통해 요구사항을 확정하고 RALPLAN-DR 계획서를 작성한 단계.
> 대화는 인터뷰 형식이라 개별 프롬프트 원문보다는 산출물에 결과가 압축되어 있음.

**최초 의도 (요약)**
> 귀여운 low-poly chibi 3D 캐릭터를 WASD로 조작하여 절차적으로 생성된 3D 마을을 탐험하고, 마을 곳곳에 흩어진 아이템을 수집하여 새로운 마을 지역(Region)을 해제하는 게임을 만들고 싶다. localStorage로 진행도를 저장하고, MeshToonMaterial + 파스텔 팔레트의 귀여운 비주얼이 핵심이다.

**딥 인터뷰 결과** (`.omc/specs/deep-interview-walk3d.md`)
- 6 rounds, 모호도 18% PASSED (threshold 20%)
- V1 범위 확정: WASD · 3인칭 카메라 · 아이템 픽업 · 수집 UI · Region 해제 · localStorage · 절차적 마을
- V2 이후: 캐릭터 커스터마이즈 · 장소 도장깨기 · 히든 비밀
- Out of Scope: 멀티플레이 · 모바일 · 서버 저장 · 전투 · NPC AI

**RALPLAN-DR 계획서** (`.omc/plans/walk3d-village-game.md`)
- Planner → Architect → Critic 합의 루프
- CRITICAL 5건 + MAJOR 3건 피드백 반영

## Phase 3 — 버그 수정 2회차 + 비주얼 폴리싱 (2026-04-20)

### P3. 스크린샷 기반 5개 이슈
> [스크린샷] 가로등이 너무 작고, 방향이 부자연스러워. 아이템 중 coin도 서서 좌우로 회전하는게 더 자연스러울 것 같고 나무도 사이즈가 너무 작은 것 같아. 그리고 고양이가 건물 뒤에 있을때는 그 건물을 안보이게 한다던지해서 무조건 고양이가 보이는 시야를 방해하지 않았으면 좋겠어. 이 작업 진행한 후에는 아까전 4개의 이슈랑 지금 말하는 이슈 문서 하나에 정리해줘

**결과**: 가로등 1.8→5.0 + 대칭형 폴백, 코인 `rotation.x=π/2`, 나무 2.5→4.0, 카메라→고양이 레이캐스트 투명화, 도로 ±7 내 소품 스킵, `FIXES.md` 생성

### P4. 코인 토러스 링 확인
> [스크린샷 — 토성 링처럼 보이는 코인] 동전에 무슨 링같은게 끼어있는데 의도한거야?

**결과**: 아니라고 답하고 토러스 제거 → 깔끔한 원반 형태

### P5. GitHub 업로드
> https://github.com/3uxeca/walk-nyang.git 에 올려줘

**결과**: 리포 초기화, `.gitignore` 설정, 초기 푸시

### P6. 하트 파티클
> 5초 이상 가만히 있을 때 계속적으로 fade in out 하면서 위로 하트들이 솟아오르게 할 수 있어?

**결과**: `HeartFX.ts` 신규 — bezier heart shape, billboard, fade 0-20%/60-100%

### P7. 하트 속도 조정
> 하트가 좀 더 천천히 떠올랐으면 좋겠어

**결과**: vy 감소로 부드럽게 조정

### P8. 맵 삭막 + 가로등 도로 배치
> [스크린샷] 가로등이 여전히 좀 작은 것 같고, 도로 한복판에 있으면 안될 것 같아. 그리고 맵이 너무 삭막해보이는데, 구름이 떠있는 하늘 같은걸 배경으로 넣거나 할 수 있어?

**결과**: `SkySystem.ts` (그라디언트 돔 + 구름 퍼프 12개), 가로등 추가 스케일 업, 도로 밴드 감지 강화

## Phase 4 — 근본 수정 + Z-fighting (2026-04-20)

### P9. 계획 수립 요청
> /plan 여전히 아직 갈 수 없는 곳에 아이템이 떠다니고있고, 도로랑 일반 지형의 퀄리티가 너무 떨어져 보이는데 어떻게 개선할 수 있을까?

**결과**: `plan --consensus --di` 파이프라인 발동. 청크 좌표계 근본 수정(`(cx+0.5)*32`), `RoadGrid.ts` 전면 재작성, 지형 셰이더 적용 계획 수립

## Phase 5 — v2 최종 버그 + 문서화 (2026-04-21)

### P11. 바닥 깜빡임
> [비디오 경로] 지금 고양이가 움직일 때 바닥면의 z-fighting인지 뭔지 깜빡이는 문제가 심한 상태인데 어떻게 해결할 수 있을까?

**결과**: `main.ts`의 중복 300×300 글로벌 바닥 제거(원인), `RoadGrid.ts`를 **non-overlap 타일 레이아웃**으로 재설계 (NS 도로 2토막, 인도/엣지 Y 레이어 분리)

### P12. 건물 투명화 전염
> [스크린샷] 고양이가 건물 뒷쪽에 가려져있을 때만 투명화 되는줄 알았는데, 뒤에 다른 건물이 있어도 투명화되는 것 같아

**결과**: 원인 발견 — GLTF `scene.clone()`이 머티리얼 공유. `AssetManager.clone()`에서 머티리얼 깊은 복제하도록 수정

### P13. 에셋 문서화
> ver.2에 쓰인 에셋들을 문서에 정리해줘 (3d kit, ogg 사운드 재생 파일 등) ogg 파일은 모두 네가 추천해줬던 freesound에서 Creative Commons 0 카테고리에 있던 것들만 가져온거야. 3d 에셋은 너가 가져온거라 난 몰라

**결과**: `ASSETS.md` 생성 — 3D 모델(Kenney/Quaternius 추정) + 사운드(Freesound CC0) + 라이선스 정리

### P14. PPT 초안
> 에셋과 UI 전면 교체 후 업데이트된 버전의 ppt 문서 만들어줘. 기술 스택, 사용 에셋 종류, 주요 업데이트 내용은 필수로 들어가되 나머지 내용은 알아서 채워줘

**결과**: `docs/walk3d-v2.md` (Marp 마크다운, 15 슬라이드)

### P15. PPTX로 변환 + 스크린샷
> 아니 업데이트 리포트 md 말고 pptx 형식으로 만들어달라고 스킬이나 플러그인 사용해서. 스크린샷도 같이 넣어서

**결과**: `python-pptx`로 `docs/walk3d-v2.pptx` 직접 생성, 스크린샷 5장 임베드 (16 슬라이드, 1.4MB), 재생성 스크립트 `build_pptx.py` 제공

### P16. OMC/멀티에이전트 내용 추가
> 개발 과정에 멀티에이전트 방식을 사용했는지, OMC의 기능들을 사용했는지(deep-interview, autopilot, ralph, plan 등), 작업 과정에서 skill이나 rule 같은 것들이 추가된 것이 있는지 등의 내용도 추가해서 업데이트 해줘. anthropic-skills:pptx 를 사용할 수 있으면 해봐.

**결과**: 3 슬라이드 추가 (OMC 파이프라인 · 멀티에이전트 테이블 · 프로젝트 메모리/룰/스킬). 총 19 슬라이드. `anthropic-skills:pptx`는 available skills에 없어 python-pptx 유지

### P17. README 작성
> git 레포에 쓸 README.md 만들어줘

**결과**: 히어로 이미지 + 배지 + 특징 + Quick Start + 기술 스택 + 프로젝트 트리 + 에셋 라이선스 + 테스트 + OMC 개발 프로세스 + 로드맵 + 기여 가이드 포함

### P18. gitignore + 커밋
