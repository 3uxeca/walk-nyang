## 목표

잠긴 지역 경계에 닿으면 "왜 못 가는지" 알려주는 짧은 토스트를 띄운다. 이전에는 플레이어 움직임만 막혀서 "왜 안 움직이지?" 싶은 혼란이 있었다.

---

## 변경 사항

### `src/ui/Toast.ts` (신규)

재사용 가능한 범용 토스트 클래스:
- 하단 중앙 위치 (`bottom: 110px`)
- `w3d-toast-in` / `w3d-toast-out` 애니메이션 (cubic-bezier spring-in)
- 동일 `key`로 쓰로틀 — 반복 충돌 시 토스트 스팸 방지
- `show(message, emoji?, key?, throttleMs = 1500)` — throttle 기본 1.5s
- `dispose()` HMR 대응

### `src/main.ts`
- `Toast` import + 모듈 레벨 변수
- `toast = new Toast()` 초기화
- locked-region 차단 지점에 `toast.show('아직 잠겨있는 지역이에요', '🔒', 'locked-region', 2000)` 호출
- HMR cleanup에 dispose 추가

---

## 쓰로틀 동작

플레이어가 경계에 기대는 행위는 거의 매 프레임 트리거되므로, 쓰로틀 없이는 토스트가 무한 재생.

| 시나리오 | 결과 |
|----------|------|
| 경계에 1회 부딪힘 | 토스트 1회 (2s) |
| 연속 1초 눌러대기 | 최초 1회만 표시, 이후 조용 |
| 2초 이상 떨어졌다 재시도 | 새 토스트 표시 |

---

## 왜 RegionUnlockFX와 별도인가

- `RegionUnlockFX`: 축하 연출 (크고 중앙 상단) — 레벨업 이벤트 발생 시
- `Toast`: 경고/안내 (작고 하단) — 매 프레임 가능한 인터랙션 피드백

톤이 달라서 같은 컴포넌트로 묶지 않았다. Wave 3에서 "수집 히스토리 타임라인" 같은 추가 알림에도 `Toast`를 재사용할 수 있다.
